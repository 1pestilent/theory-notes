# Schedulers



Scheduler, или планировщик - управляет таймерами запуска задач

Job, или задача - метод, который нужно вызвать

Тригер - условие выполнения задачи



#### Виды планировщиков:



##### Локальный

Это базовый инструмент Java. Они просты, но в микросервисах опасны без допила, поскольку не знают о существовании других реплик.



* @Scheduled - аннотация
* TaskScheduler - интерфейс



##### Распределенный



Гарантируют, что в кластере задачу выполнит только один сервис. Основной иструмент для распределенного планировщика - блокировщик ShedLock. Он дополнительно использует Redis/Postgres и создает таблицу, которую использует для координации инстансов.



|name|lock\_until|locked\_at|locked\_by|
|-|-|-|-|
|Уникальное имя задачи|До какого момента задача заблокирована.|Когда замок был захвачен.|Кем захвачен замок. Обычно туда пишется IP сервера или имя пода в Kubernetes|

Инфраструктурный

Логика планирования за пределами Java-кода. YAML расписание и образ контейнера, который выполняет какие-то манипуляции. Куб сам в нужное время поднимает Под, он отрабатывает задачу и умирает.
